# 바인그룹 실시간 온라인 AI 교육 2차수 강의 대본

- 주제: 데이터 처리와 자동화 with AI
- 강의 목표: 흩어진 업무 데이터를 분석 가능한 구조로 정리하고, 계산·피벗·인사이트가 포함된 Excel 파일을 완성한다.
- 최종 제출물: `02_최종_분석결과_이름.xlsx`
- 강의 흐름: 문제 정의 → 데이터 구조 읽기 → 전처리 → 계산 열 → 피벗 → 인사이트 → Excel 제출

## 1. 도입과 오늘의 방향

### 1페이지 · 표지

안녕하세요. 오늘은 AI에게 데이터를 바로 분석시키는 방법보다, AI가 제대로 읽고 계산할 수 있는 데이터 구조를 만드는 방법을 다룹니다. 수업이 끝나면 계산 데이터와 피벗 결과, 인사이트가 들어 있는 Excel 파일 한 개를 완성하게 됩니다.

### 2페이지 · 강사 소개

저는 이유진입니다. 오늘은 기능을 많이 설명하기보다 실제 업무 데이터를 어떤 순서로 정리해야 반복해서 쓸 수 있는지에 집중하겠습니다.

### 3페이지 · 2주차 방향

AI 모델을 고르기 전에 먼저 볼 것은 데이터입니다. 같은 AI를 써도 데이터가 흩어져 있거나 형식이 섞여 있으면 결과가 달라지므로, 오늘은 AI가 읽을 수 있는 상태를 먼저 만들겠습니다.

### 4페이지 · 사전 설문 요약

사전 설문을 보면 어려움은 도구 부족보다 데이터 취합, 형식 통일, 비교와 보고의 반복에 있었습니다. 오늘 실습은 이 반복 작업을 줄이는 기준을 만드는 데 맞춰져 있습니다.

### 5페이지 · 실제 고민으로 시작되는 오늘의 실습

오늘 예시는 수강생 여러분이 실제로 남긴 고민에서 출발했습니다. 매출과 상담 데이터를 모으고, 형식을 맞추고, 비교 가능한 표로 만드는 과정을 차례대로 해보겠습니다.

### 6페이지 · 2주차에서 실제로 하는 것

오늘의 목표는 분석 결과를 멋지게 보여주는 것이 아닙니다. 계속 사용할 수 있도록 기준이 잡힌 데이터를 만들고, 계산 열과 피벗 기준까지 남기는 것입니다.

### 7페이지 · 현재 나의 업무 데이터는 어디에 있나요?

5분 정도 떠올려보겠습니다. 내가 자주 쓰는 데이터는 어느 폴더에 있고, 이메일·메신저·설문·시트 중 어디에서 들어오나요? 데이터의 위치와 출처를 알아야 한 표로 묶을 기준도 만들 수 있습니다.

### 8페이지 · 2~3차수 로드맵

2주차는 데이터 처리와 자동화, 3주차는 데이터 분석과 시각화입니다. 오늘은 데이터를 계산 가능한 상태로 만들고, 다음 차수에서는 그 데이터를 본격적으로 비교하고 시각화합니다.

### 9페이지 · 연습용 샘플 데이터 폴더링

1주차에서 배운 폴더 구조를 그대로 사용합니다. 원본, AI 정리 결과, 실습 캡처를 분리해두면 작업 중 문제가 생겨도 원본으로 돌아갈 수 있습니다.

### 10페이지 · 오늘 다룰 실습 예시 3가지

오늘은 세 가지 실습만 이어서 진행합니다. 실습 1은 구조 읽기, 실습 2는 계산 열과 피벗, 실습 3은 인사이트와 Excel 최종 파일 정리입니다.

### 11페이지 · 1차수 복습

폴더 구조와 파일명 규칙은 단순 정리가 아니라 업무 자동화의 출발점입니다. 같은 위치와 같은 이름 규칙을 유지해야 다음 작업에서도 파일을 다시 찾고 재사용할 수 있습니다.

## 2. 핵심 이론과 전처리

### 12페이지 · 잠깐, 핵심 이론 점검

실습 전에 꼭 필요한 개념만 짧게 보겠습니다. 이론을 길게 외우기보다 곧바로 다음 실습에서 써보는 것이 목적입니다.

### 13페이지 · 바인그룹 데이터 업무 운영 프레임워크

데이터 업무를 질문, 구조 읽기, 전처리, 계산과 피벗, 인사이트의 다섯 단계로 나눕니다. AI에게 한 번에 전부 맡기지 않고 단계별 결과를 확인하는 방식입니다.

### 14페이지 · AI 데이터 업무는 한 번 묻고 끝나는 작업이 아닙니다

좋은 결과는 질문만 잘 쓴다고 나오지 않습니다. 데이터의 맥락을 알려주고, 처리 기준을 정하고, 마지막에는 사람이 숫자를 검산해야 합니다.

### 15페이지 · 문제 정의

먼저 오늘 보고 싶은 질문 하나를 고르겠습니다. ‘어느 센터가 문제인가’보다 ‘매출은 높은데 전환이 낮은 센터는 어디인가’처럼 비교 기준이 들어간 질문이 좋습니다.

### 16페이지 · 데이터 종류

행과 열로 바로 계산할 수 있는 자료는 정형 데이터입니다. 후기, 문의, 회의록처럼 문장을 읽어야 하는 자료는 비정형 데이터이며, 먼저 분류·요약 표로 바꾸는 작업이 필요합니다.

### 17페이지 · 내 자료는 어디서 시작해야 할까요?

정형 데이터라면 열 의미와 형식부터 점검하고, 비정형 데이터라면 분류 기준과 요약 항목부터 만듭니다. 화면의 프롬프트는 각 자료 유형에 맞춰 복사해 사용할 수 있습니다.

### 18페이지 · 전처리를 해봅시다

이제 실제로 데이터를 고쳐보겠습니다. 다음 몇 장에서는 왜 전처리가 필요한지 확인하고, 연습용 원본에서 문제를 직접 찾습니다.

### 19페이지 · 전처리란 무엇인가

전처리는 표를 예쁘게 꾸미는 일이 아닙니다. 날짜, 이름, 단위, 빈칸, 중복 같은 기준을 맞춰 계산과 비교가 틀리지 않게 만드는 작업입니다.

### 20페이지 · 전처리를 실제로 한 번 해봅시다

오른쪽의 데이터 다운로드 링크에서 연습용 샘플을 받습니다. 원본 파일은 그대로 보관하고 복사본에서 형식 통일, 중복 확인, 개인정보 제거를 진행하겠습니다.

### 21페이지 · 실습용 원본 예시

표에서 네 가지 문제를 찾아보겠습니다. 같은 사람이나 센터가 다른 표현으로 적혀 있고, 매출 단위가 섞여 있으며, 메모에는 주소와 전화번호 같은 민감정보가 들어 있습니다.

### 22페이지 · 전처리 품질은 6개 키워드로 확인합니다

전처리 품질은 형식, 단위, 누락, 중복, 표준화, 보안으로 확인합니다. 이 여섯 가지를 체크하면 피벗과 수식에서 생기는 대부분의 오류를 미리 줄일 수 있습니다.

### 23페이지 · AI에게 먼저 알려줘야 하는 것

파일만 올리면 AI는 열의 뜻을 추정할 수밖에 없습니다. 데이터의 출처, 기간, 한 행의 기준, 주요 열의 정의, 보고 싶은 질문을 함께 알려줘야 해석이 안정됩니다.

### 24페이지 · AI에게 맡길 일과 사람이 볼 일

AI는 열 설명, 전처리 후보, 수식 초안, 피벗 구성을 빠르게 제안할 수 있습니다. 하지만 실제 숫자가 맞는지, 빈칸을 어떻게 볼지, 결론이 타당한지는 사람이 결정해야 합니다.

### 25페이지 · 개인정보는 이렇게 바꿔서 넣습니다

이름, 연락처, 주소, 계약번호처럼 개인을 식별할 수 있는 값은 삭제하거나 가명으로 바꿉니다. 실제 원본 대신 구조가 같은 샘플로 먼저 테스트하는 습관이 중요합니다.

## 3. 실습 1 — 데이터 구조 읽기

### 26페이지 · 데이터 구조 읽기

첫 번째 실습은 분석이 아니라 진단입니다. AI에게 파일을 올리고 열의 의미, 데이터 타입, 확인 필요 항목, 가능한 업무 질문을 뽑아보겠습니다.

### 27페이지 · ChatGPT와 Gemini, 언제 무엇을 쓸까

파일 전체 구조를 읽고 기준을 설계할 때는 ChatGPT가 편리하고, Google Sheets 안에서 여러 행을 반복 처리할 때는 Gemini가 유리할 수 있습니다. 오늘은 둘 중 익숙한 도구 하나만 선택하면 됩니다.

### 28페이지 · 실습 1에서 할 일

실습 파일을 내려받고, AI에 업로드한 뒤, 구조 읽기 프롬프트를 실행합니다. 결과물은 열 설명표, 확인 필요 항목, 내 업무와 가까운 분석 질문 한 개입니다.

### 29페이지 · 실습 파일 다운로드

`00_RAW_매출실적_가상.csv`를 내려받아 원본 폴더에 저장합니다. 이 파일을 실습 1에서는 AI에 업로드하고, 실습 2에서는 Excel 또는 Google Sheets로 열어 이어서 사용합니다.

### 30페이지 · 데이터 구조 읽기 프롬프트

파일을 첨부한 상태에서 화면의 프롬프트를 복사해 실행합니다. 결과에서 AI가 열을 잘못 이해한 부분이나 확인이 필요한 부분이 있는지 먼저 보겠습니다.

### 31페이지 · ChatGPT로 실습 시작하기

ChatGPT를 선택했다면 새 대화에서 플러스 버튼으로 CSV를 첨부합니다. 파일이 올라간 것을 확인한 뒤 프롬프트를 붙여넣고 전송합니다.

### 32페이지 · Gemini로 실습 시작하기

Gemini를 선택했다면 같은 방식으로 파일을 첨부하고 프롬프트를 실행합니다. ChatGPT와 Gemini를 둘 다 할 필요는 없으며, 결과 표와 확인 필요 항목만 확인하면 됩니다.

### 33페이지 · 실습 1 실제 결과 화면

정상 결과에서는 확인 영역, 검산 기준, 현재 결과, 권장 조치가 표로 정리됩니다. 이 가운데 실제 원본을 보고 사람이 확인해야 하는 항목 하나를 고릅니다.

## 4. 실습 2 — 계산 열과 피벗

### 34페이지 · 계산 열과 피벗 설계

두 번째 실습에서는 전환율, 환불률, 등록당 매출처럼 비교에 필요한 계산 열을 만들고, 피벗으로 센터와 채널을 바꿔 보겠습니다.

### 35페이지 · 피벗 정의

피벗은 원본 데이터를 바꾸는 기능이 아니라 같은 데이터를 다른 기준으로 요약하는 기능입니다. 행에는 비교 대상, 열에는 세부 기준, 값에는 합계나 평균을 낼 숫자, 필터에는 볼 범위를 넣습니다.

### 36페이지 · 실습 2에서 할 일

CSV 파일을 Excel 또는 Google Sheets로 열고 AI에게 계산 공식을 요청합니다. 첫 몇 행을 손으로 검산한 다음 피벗의 행, 열, 값, 필터를 배치합니다.

### 37페이지 · 계산 열·피벗 설계 프롬프트

파일을 AI에 업로드하고 화면의 프롬프트로 전환율, 환불률, 등록당 매출 공식과 피벗 구성을 받습니다. AI는 공식을 제안하고, 실제 적용과 검산은 Excel 또는 Sheets에서 합니다.

### 38페이지 · 실제 화면: 전환율과 환불률 계산

전환율은 등록수 나누기 상담신청수, 환불률은 환불건수 나누기 등록수입니다. 첫 행을 계산기로 직접 계산해 수식 결과와 같은지 확인한 뒤 전체 행에 적용합니다.

### 39페이지 · 피벗테이블을 직접 만들어 봅니다

원본 표 전체를 선택하고 삽입 메뉴에서 피벗테이블을 만듭니다. 행은 센터, 열은 채널, 값은 매출 합계, 필터는 월로 놓고 값이 개수가 아닌 합계로 계산되는지 확인합니다.

### 40페이지 · 피벗 결과를 질문으로 바꿉니다

피벗이 완성되면 높은 값만 찾지 말고 차이가 나는 이유를 질문으로 바꿉니다. 전환율과 환불률을 다시 계산해 사실, 해석, 확인 필요, 다음 질문으로 구분합니다.

## 5. 실습 3 — 인사이트와 Excel 최종 파일

### 41페이지 · 인사이트와 전처리 데이터 정리

마지막 실습은 계산 결과를 업무 판단으로 연결하고 Excel 파일 한 개로 정리하는 단계입니다. 수치 근거, 확인 필요, 다음 액션을 함께 남깁니다.

### 42페이지 · 실습 3 데이터 파일 준비

실습 2에서 저장한 파일이 있으면 그대로 사용합니다. 없다면 원본 CSV를 다시 내려받아 계산 열과 피벗 결과를 만든 뒤 `02_최종_분석결과_이름.xlsx`로 저장합니다.

### 43페이지 · 인사이트 정의

인사이트는 숫자 자체가 아니라 숫자로 만든 판단입니다. 사실을 확인하고, 차이를 해석하고, 단정할 수 없는 부분을 표시한 뒤 다음 액션까지 연결합니다.

### 44페이지 · 인사이트 프롬프트

Excel 파일과 함께 화면의 프롬프트를 입력합니다. 결과는 핵심 판단, 수치 근거, 확인 필요, 다음 액션, 추천 차트로 받고 `03_인사이트` 시트에 붙일 수 있는 표 형태로 정리합니다.

### 45페이지 · 실제 화면: 수치에서 다음 액션 만들기

결과 화면에서 먼저 숫자가 원본 계산과 맞는지 봅니다. 사실과 해석이 구분되어 있는지, 확인 필요 항목이 남아 있는지, 다음 액션이 구체적인지 확인합니다.

### 46페이지 · 업무 질문별 차트 선택

차트는 질문에 따라 선택합니다. 대상 비교는 막대, 시간 변화는 꺾은선, 구성 비율은 누적 막대, 두 값의 관계는 산점도가 기본입니다.

### 47페이지 · Excel 최종 결과물

최종 파일에는 `01_계산데이터`, `02_피벗결과`, `03_인사이트`, `04_차트` 시트가 있어야 합니다. 수식이 살아 있는 `.xlsx` 형식으로 저장하고 PDF나 캡처 이미지만 제출하지 않습니다.

### 48페이지 · AI 데이터 분석 검수 기준

제출 전에 보안, 단위, 단정 표현, 원본 보존을 확인합니다. 특히 AI가 제안한 원인을 사실처럼 확정하지 않았는지와 실제 개인정보가 남아 있지 않은지를 봅니다.

### 49페이지 · 오늘 이후 이어지는 작업

오늘 만든 파일은 다음 차수의 출발점입니다. 3차수에서는 같은 계산 데이터와 피벗 결과를 사용해 데이터 분석과 시각화를 더 깊게 진행합니다.

### 50페이지 · 오늘 정리

오늘은 원본을 보존하고, 계산을 검산하고, 판단을 근거와 함께 남겼습니다. 분석을 끝낸 것이 아니라 앞으로 반복해서 분석할 수 있는 기준을 만든 것입니다.

### 51페이지 · Padlet 최종 제출

마지막으로 `02_최종_분석결과_이름.xlsx` 파일을 Padlet에 업로드합니다. 네 개 시트가 있는지 확인하고, 수식·피벗·해석 중 가장 어려웠던 지점을 한 줄로 함께 남겨주세요.
