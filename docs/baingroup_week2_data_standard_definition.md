# 바인그룹 2차수 데이터 기준정의서

## 1. 목적

여러 채널의 데이터를 한 표에서 비교하려면, 데이터를 붙이기 전에 먼저 기준을 정해야 합니다.
이 문서는 바인그룹 2차수 실습에서 사용할 마스터 표의 규칙을 정리한 기준정의서입니다.

오늘의 원칙은 다음 세 가지입니다.

- 원본 파일은 수정하지 않고 보존합니다.
- 서로 다른 자료가 만날 기준 키를 먼저 정합니다.
- 개인정보와 사내 기밀은 삭제하거나 가짜 샘플로 치환한 뒤 AI에 질문합니다.

## 2. 권장 폴더 구조

```text
00_원본자료
├─ 센터별_매출
├─ 상담_문의
├─ 후기_피드백
└─ 운영_공지

01_정리기준
├─ 데이터_기준정의서.md
├─ 열_매핑표.xlsx
└─ 표준값_사전.xlsx

02_AI정리결과
├─ 전처리_후보
├─ 계산열_피벗기준
└─ 보고서_초안
```

## 3. 마스터 표 핵심 기준

| 기준 | 정의 | 예시 |
|---|---|---|
| 기준 월 | 데이터를 비교할 월 단위 | `2026-07` |
| 센터 | 센터명을 표준값으로 통일 | `강남`, `강남센터` → `강남센터` |
| 과정 | 교육 과정 또는 상품 분류 | `초등`, `중등`, `렌탈`, `상담` |
| 채널 | 데이터가 발생한 경로 | `홈페이지`, `블로그`, `광고`, `소개`, `센터 입력` |
| 지표 | 비교할 숫자 항목 | 상담신청수, 등록수, 매출, 환불건수, 방문자수 |
| 출처 | 데이터가 온 파일/시스템 | 파일명, 시트명, 시스템명 |
| 확인 상태 | 누락/중복/이상값 처리 상태 | 정상, 확인필요, 제외, 익명화완료 |

## 4. 기본 열 정의

| 열 이름 | 의미 | 입력 규칙 | 확인할 점 |
|---|---|---|---|
| `month` | 기준 월 | `YYYY-MM` | 월/일 단위가 섞이지 않았는지 확인 |
| `center` | 센터명 | 표준값 사전 기준 | 강남/강남센터처럼 다른 표기 통일 |
| `course` | 과정명 | 표준 과정명 | 과정명이 빈칸이면 확인 필요 |
| `channel` | 유입/자료 채널 | 홈페이지, 블로그, 광고, 소개 등 | 채널명 표기 통일 |
| `lead_count` | 상담신청수 | 숫자 | 텍스트나 쉼표가 섞였는지 확인 |
| `paid_count` | 등록수 | 숫자 | 상담신청수보다 큰 경우 확인 |
| `revenue` | 매출 | 원 단위 숫자 | 원/만원/백만 단위 혼합 확인 |
| `refund_count` | 환불건수 | 숫자 | 빈칸이 0인지 미입력인지 확인 |
| `source_file` | 원본 파일명 | 파일명 그대로 | 오류 추적을 위해 유지 |
| `quality_flag` | 품질 확인 상태 | 정상, 확인필요, 제외 | AI가 제안하고 사람이 확정 |

## 5. 표준값 사전

| 항목 | 원본 표현 | 표준값 |
|---|---|---|
| 센터명 | 강남, 강남센터, 강남점 | 강남센터 |
| 센터명 | 분당, 분당센터 | 분당센터 |
| 채널 | HP, 홈페이지, 웹사이트 | 홈페이지 |
| 채널 | 블로그, 네이버블로그 | 블로그 |
| 상태 | 등록완료, 결제완료, paid | 등록 |
| 상태 | 환불, 취소, refund | 환불 |

## 6. 전처리 체크리스트

- [ ] 원본 파일과 작업 파일을 분리했는가?
- [ ] 기준 월, 센터, 과정, 채널 열이 있는가?
- [ ] 날짜 형식이 `YYYY-MM-DD` 또는 `YYYY-MM`으로 통일되었는가?
- [ ] 매출, 건수, 비율의 단위가 섞이지 않았는가?
- [ ] 누락값이 0인지 미입력인지 구분했는가?
- [ ] 중복 행 후보를 표시했는가?
- [ ] 실명, 연락처, 계약번호, 상세 주소, 민감 메모를 제거했는가?
- [ ] AI가 만든 수식은 표본 3~4행으로 검산했는가?

## 7. 보안 데이터 처리 기준

AI에 원본 파일을 바로 올리지 않습니다. 다음 순서로 샘플을 만든 뒤 질문합니다.

1. 원본 파일은 `00_원본자료`에 보존합니다.
2. 작업용 복사본을 만듭니다.
3. 실명, 연락처, 주민번호, 계약번호, 상세 주소, 민감 상담 메모를 삭제합니다.
4. 고객명은 `고객001`, `고객002`처럼 가명 처리합니다.
5. 상세 매출이 민감하면 구간값으로 바꿉니다. 예: `1천만~2천만`
6. AI에는 열 이름, 단위, 기간, 분석 질문만 설명합니다.
7. AI가 제안한 수식과 절차는 실제 파일에 직접 적용하고 표본 검산합니다.

## 8. AI에 요청할 프롬프트

```text
아래 데이터는 실제 개인정보를 제거한 샘플입니다.
이 표를 기준으로 데이터 기준정의서를 만들어줘.

포함할 항목:
1. 각 열의 의미
2. 표준화가 필요한 값
3. 누락값/중복값/이상값 확인 기준
4. 개인정보나 사내 기밀로 보이는 열
5. 전환율, 환불률처럼 만들 수 있는 계산 열
6. 피벗테이블로 볼 수 있는 비교 기준

결과는 표로 작성해줘.
```

## 9. 오늘 실습 최종 제출 기준

완성본이 아니어도 제출 가능합니다. 아래 중 하나만 남겨도 됩니다.

- 데이터 기준정의서 초안
- 열 매핑표
- 표준값 사전
- 전처리 후보 목록
- 계산 열 또는 피벗 기준
- AI 프롬프트와 결과 캡처
- 막힌 단계와 질문
